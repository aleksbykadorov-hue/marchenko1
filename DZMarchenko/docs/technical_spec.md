# Техническое задание (ТЗ): чат-приложение с групповыми чатами

## 1. Назначение системы

Система предназначена для обмена сообщениями между пользователями, включая **создание групповых чатов**, отправку/получение сообщений, аутентификацию и базовое управление сущностями.

## 2. Роли

- **Пользователь**: регистрируется, входит в систему, создает чаты, отправляет сообщения.

## 3. Функциональные требования

### 3.1 Пользователи и аутентификация (`user-service`)

- Регистрация пользователя (email + password)
- Авторизация по логину/паролю, получение JWT
- Получение профиля текущего пользователя по JWT

### 3.2 Чаты и сообщения (`chat-service`)

- Создание группового чата
- Получение списка чатов пользователя
- Добавление участника в чат
- Отправка сообщения в чат
- Получение сообщений чата (постранично)
- Удаление сообщения автором (soft delete) либо администратором чата (упрощенно: автор)

### 3.3 Уведомления (`notification-service`)

- Эндпоинт для приема запроса на уведомление (демо)
- Логирование/хранение факта отправки (упрощенно)

## 4. Нефункциональные требования

- **Безопасность**: пароли хранятся только в хэшированном виде (bcrypt)
- **JWT**: срок жизни access token, секреты только через переменные окружения
- **Надежность**: валидация входных данных (Pydantic)
- **Документация**: OpenAPI (`/docs`) для каждого сервиса
- **Тестируемость**: юнит-тесты на основные сценарии (auth + CRUD)

## 5. Сущности данных (логическая модель)

- **User**: id, email, password_hash, created_at
- **Chat**: id, title, owner_user_id, created_at
- **ChatMember**: chat_id, user_id, role
- **Message**: id, chat_id, author_user_id, text, created_at, deleted_at?
- **Notification**: id, type, payload, created_at, status

## 6. API (кратко)

Подробно — в Swagger UI каждого сервиса.

- `user-service`
  - `POST /auth/register`
  - `POST /auth/login`
  - `GET /me`

- `chat-service`
  - `POST /chats`
  - `GET /chats`
  - `POST /chats/{chat_id}/members`
  - `POST /chats/{chat_id}/messages`
  - `GET /chats/{chat_id}/messages`
  - `DELETE /messages/{message_id}`

- `notification-service`
  - `POST /notify`

## 7. Критерии приемки (минимум)

- Сервисы поднимаются через `docker compose up --build`
- Регистрация + логин возвращает JWT
- С JWT можно создать чат, добавить участника, отправить сообщение и получить список сообщений
- Есть `pytest` тесты, проходящие локально
