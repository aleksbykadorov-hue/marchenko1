Чат-приложение: Система обмена сообщениями с возможностью создания групповых чатов.
Проектирование архитектуры: Нужно выбрать архитектуру приложения — монолитную или микросервисную. Требуется спроектировать основные компоненты приложения (API, база данных, сервисы) и описать взаимодействие между ними. Необходимо также описать архитектурные решения, включая выбор фреймворков и технологий (например, Flask или FastAPI для API, PostgreSQL или MongoDB для базы данных). Важно применять архитектурные паттерны (например, MVC, Factory, Singleton) для организации кода и архитектуры.
Ожидаемый результат:
Архитектурная диаграмма приложения.
Описание выбранных технологий, паттернов и компонентов системы.
Техническое задание с описанием функционала и требований к системе.
Этап 3: Реализация основного API и контейнеризация (3 недели)
Цель:
Необходимо реализовать базовое API для приложения, настроить контейнеризацию с использованием Docker и протестировать основные функциональные возможности.
Задачи:
Реализация API: Нужно создать основные эндпоинты для работы с сущностями приложения (например, создание, редактирование, удаление и отображение задач, постов или товаров). Реализовать базовые операции CRUD (создание, чтение, обновление, удаление) для основной сущности. Внедрить систему аутентификации и авторизации, например, с использованием JWT.
Тестирование API: Написать юнит-тесты для каждого из API-эндпоинтов с использованием библиотеки pytest. Создать тесты для проверки функционала CRUD и аутентификации.
Контейнеризация: Создать Docker-образ для приложения. Настроить Docker Compose для контейнеризации приложения и базы данных, если это необходимо.
Ожидаемый результат:
Рабочее API с базовыми функциями (CRUD, аутентификация).
Docker-образ для приложения.
Набор юнит-тестов для API.
Этап 4: Разработка и интеграция микросервисов (3 недели)
Цель:
Необходимо разработать микросервисы, которые будут интегрированы в приложение для расширения его функциональности.
Задачи:
Разработка микросервисов: Нужно разработать один или несколько микросервисов для выполнения конкретных функций в рамках приложения:
Микросервис для работы с пользователями (регистрация, аутентификация).
Микросервис для обработки платежей (интеграция с системой оплаты).
Микросервис для отправки уведомлений (например, email-уведомления или push-уведомления).
Интеграция микросервисов: Микросервисы должны взаимодействовать с основным приложением через API. Для интеграции микросервисов необходимо настроить их взаимодействие с помощью REST API или других средств (например, очереди сообщений).
Документация микросервисов: Нужно подготовить документацию для каждого микросервиса с описанием его функционала и взаимодействия с другими компонентами системы.
Ожидаемый результат:
Разработанные и интегрированные микросервисы.
Документация для микросервисов, включающая описание их функционала и API.
Этап 5: Оптимизация, тестирование и развертывание (3 недели)
Цель:
Необходимо провести оптимизацию приложения, протестировать его компоненты и развернуть приложение в облаке.
Задачи:
Оптимизация приложения: Нужно оптимизировать работу API и микросервисов для повышения производительности. При необходимости реализовать кеширование данных или другие оптимизационные техники.
Тестирование: Провести интеграционные тесты для проверки взаимодействия всех компонентов системы. Убедиться, что все микросервисы правильно интегрированы и приложение работает стабильно.
Развертывание приложения: Развернуть приложение в облаке (например, на платформе Heroku, AWS или GCP). Настроить CI/CD для автоматической сборки и развертывания приложения.
Ожидаемый результат:
Оптимизированное приложение с улучшенной производительностью.
Интеграционные тесты.
Развернутое приложение в облаке с настроенным CI/CD.
